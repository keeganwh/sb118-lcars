1. Overview

LCARS is a local, offline writing tool purpose-built for Play By Email (PBEM) roleplay at Starbase 118. It provides a rich-text word processor with script-format support, organized document storage, a character roster, and a clean copy/paste workflow optimized for posting sims to the wiki and mailing list.

Everything runs inside a single HTML file — no installation, no account, no internet connection required (except to load the Droid Sans font on first use). All your writing is saved privately to your own browser.

Version 2 adds six major features: a Command Dashboard, full-text search, a Character Manifest with profiles and pictures, sim templates, revision snapshots, and a one-click Copy Post button.

2. Getting Started

Open LCARS.html in any modern browser (Chrome, Edge, Firefox, or Safari). You can open it directly from your file system — no web server is needed.

Tip: Bookmark the file for quick access. Your data persists automatically via browser storage between sessions.

To share LCARS with a friend, send them the single LCARS.html file. They need nothing else. Their data will be completely separate from yours.

Tip: Export your data regularly as a backup (⬇ Export in the header). Browser storage can be wiped if you clear browsing data. The .lcars file is your permanent save.

3. The Interface

Header Bar

The amber LCARS bar at the top contains:

  • + Sim, + Scene, + Mission — quick-create new items
  • ◉ Manifest — opens the Character Manifest overlay
  • ⬇ Export / ⬆ Import — save/restore your data file
  • ⚙ — opens the Settings & Preferences popover

Left Sidebar — SIMS

The sidebar displays your full content tree, organized by Year → Mission → Scene → Sim. Click any item to open it; click the ▼ caret to collapse a branch.

The 🔍 button at the top of the sidebar activates full-text search (see Section 13). The ◀ button collapses the sidebar to a sliver; drag its right edge to resize it freely.

The 📁 Archived button at the bottom toggles visibility of archived items, which are hidden by default.

Right-click any item in the tree for its context menu — rename, change status, set tags, or delete.

Editor Panel (Centre)

When a sim is open, the centre shows the sim header (a two-row strip containing the title, controls, and the full writing toolbar) followed by the writing area. When no sim is open, the Dashboard is shown instead.

Right Panel — Characters

The Characters panel automatically lists every character name detected in the current sim. Click a name to mark it as one of your characters (shown in blue). Right-click a name for additional options — rename, fix spelling, or manage the "different characters" confirmation.

The ▶ button collapses the Characters panel; drag its left edge to resize it.

Status Bar

The bar at the bottom shows the current sim's title, live word count, and zoom controls (− / +).

4. Organizing Your Writing — The Hierarchy

Content is organized in a four-level tree:

▼ 2401                    (Year)
  ▼ The Wrath of Omega   (Mission)
    ▼ Briefing Room Alpha (Scene)
      JP: The Gathering Storm (Sim)
      Solo: Into the Dark
  • Year — Groups missions by stardate year. Created automatically from the year you enter when making a mission. Right-click a year to add a mission or delete the entire year (double confirmation required).
  • Mission — The top-level story arc. Created with + Mission. Click the mission name in the sidebar to open a Mission Detail view showing all its scenes and sims.
  • Scene — A subdivision of a mission (a location, thread, or subplot). Created with + Scene. Click the scene name to open a Scene Detail view.
  • Sim — An individual post or joint post. Created with + Sim. Sims can belong directly to a mission or be placed inside a scene.

Creating Items

Use the header buttons or right-click any existing item to create a child item beneath it. When creating a new sim, you can optionally start it from a saved template (see Section 14).

Renaming & Deleting

Right-click any mission or scene to rename or delete it. Deleting cascades downward — a confirmation dialog lists exactly what will be removed. Right-clicking a Year header offers "Delete Year…" which requires two confirmations before permanently removing all content for that year.

Detail Views

Clicking a mission name or scene name in the sidebar opens a detail view showing stats, a list of child items, and status information. Click any sim title within the detail view to open it for editing.

Copy & Paste

When you copy text from the editor — either by selecting manually or using the 📋 Copy button — LCARS automatically strips all colours, backgrounds, and decorative class markers. Only bold, italic, links, and indentation are preserved. This produces clean output ready to paste into your email client or posting platform.

Auto-save

Changes are saved automatically approximately 1.2 seconds after you stop typing. The date in Row 1 of the header confirms the last save time.

Zoom

The − and + buttons in the status bar zoom the editor text without affecting the surrounding UI. Zoom level is saved between sessions.

6. Script Markers

LCARS recognises three special markup patterns used in SB118 sims and renders them with subtle visual highlights to aid readability while writing. The highlighting is display only — markers are stored and copied as plain text.

SyntaxWhat it marksAppearance
::text:: Action / narration beat ::tapping the console::
=/\=text=/\= Comms / radio transmission =/\= Bridge to Hopper. =/\=
oO text Oo Internal thought oO What was that? Oo

Markers are applied automatically as you type (with a short delay). The toolbar buttons in Row 2 toggle each marker type on or off — useful if you find certain highlights distracting for a particular session.

Marker highlighting does not affect the stored content or what gets copied to the clipboard.

7. Character Detection

LCARS automatically detects character names by looking for lines that begin with a capitalized name followed by a colon, matching the standard SB118 script format:

Hopper: All right people, let's go.
Martin/Hopper: ::exchanging a glance::
oO I've got a bad feeling about this. Oo

All detected names appear in the Characters panel on the right. Click a name to toggle it as one of your characters (shown in blue). Your designations are saved per-sim and also carry forward — if you mark a character in one sim, they'll be recognised automatically in others.

Alias Detection

If a character in the Character Manifest has aliases defined (see Section 12), LCARS will also detect those alias forms in the sim — including names with initials and periods that the standard rule can't match on its own. For example, if Robin Hopper has aliases Hopper and R. Hopper, then a line starting with R. Hopper: will be detected and automatically attributed to Robin Hopper's record.

Auto-tagging

When a sim is saved or opened, LCARS automatically adds any detected character name to the sim's "my characters" list if it — or any of that character's aliases — has been tagged as yours before. You do not need to manually tick every character in every sim; the system resolves alias chains silently in the background.

Similar Name Warning

If a character name in the current sim closely resembles a name used in another sim — suggesting a possible spelling inconsistency — a ⚠ warning appears next to the name. Click it to review the options:

  • Use "[similar name]" — replace the spelling in this sim to match the other
  • Keep spelling — dismiss the warning for now
  • They're different — mark these as two distinct characters; the warning won't appear again for this pair

Character Context Menu

Right-click any character name in the Characters panel for additional options:

  • ✏ Rename — replace the character's name throughout the current sim
  • ⟳ Use "[other name]" — fix the spelling to match the similar name
  • ✓ They are different people — confirm the pair and suppress the warning
  • ↩ Un-mark "≠ …" — remove a previously confirmed "different characters" pair
  • Mark as my character / Remove from my characters — toggle ownership

8. Posting & Status

Status

Every sim, scene, and mission has a status, shown as a coloured dot in the sidebar:

  • Active — work in progress
  • Complete — finished and posted
  • Archived — hidden from the main view

Change a sim's status using the dropdown in the doc header. Right-click a mission or scene to change its status — this cascades down to all child scenes and sims automatically.

Posting Date

Click 📅 Post in the sim header to record the date a sim was posted. A date picker opens. Selecting a date:

  • Records the date and displays it in the header as Posted [date] in green, replacing the "Saved" label
  • Automatically marks the sim as Complete

To update the date, click 📅 Repost. Switching status back to Active clears the posted date.

9. Tags & Mission Types

Two independent tagging systems categorize your content at a glance.

Mission Type (set on Mission)

Right-click a mission and choose a tag from the context menu:

  • Mission — active mission storyline
  • Shore Leave — off-duty or shore leave period
  • Academy — Academy training sim (enables Academy Mode — see Section 10)
  • Untagged — general / regular (default)

Post Type (set on Sim)

Use the Sim type dropdown in the sim header (Row 2):

  • JP — Joint Post (written collaboratively)
  • Solo — Solo post (single author)
  • Untagged (default)

Tags appear as coloured pills next to items in the sidebar and in detail views.

10. Academy Mode

When a mission is tagged Academy, all sims within it operate in Academy Mode. This reflects the plain-text convention used in Academy training sims.

In Academy Mode:

  • An amber banner appears at the top of the editor — ACADEMY MODE — Plain text only. Rich formatting is disabled.
  • Bold, italic, strikethrough, indentation, and links are all disabled for that sim
  • Existing rich formatting is automatically stripped from any sim opened in an Academy mission
  • Script marker highlighting (::, =/\=, oO) still works
Academy Mode applies at the mission level — tagging a mission as Academy affects all its scenes and sims.

11. Dashboard New in v2

When no sim is open, the Command Dashboard is displayed in the centre panel. It gives you an at-a-glance overview of your writing activity.

Stats Bar

The row of counters at the top shows your total missions, total sims, posted sims, and cumulative word count across all content.

In Progress

Lists up to 10 sims with Active status, sorted by the most recently edited. Each row shows the sim title (click to open), mission name, word count, and last edited date.

Recently Posted

Lists the 5 most recently posted sims, sorted by posted date. Click any title to open it.

Quick Actions

The + New Sim and + New Mission buttons provide the same functionality as the header buttons, without having to move your mouse to the top of the screen.

Tip: Click ✕ in the sim header to close a sim and return to the Dashboard at any time.

12. Character Manifest New in v2

The ◉ Manifest button in the header opens the full-screen Character Manifest — a profile database for all your characters, with stats drawn automatically from your sims.

Character List (Left Pane)

The left pane shows all manifest characters. Each entry displays the character's avatar (photo or coloured initials), name, rank with pip visualization, division colour dot, and character type badge.

Filtering by Type

A row of filter chips at the top of the list narrows it to a specific character type:

  • Primary — your main player character(s)
  • Secondary — secondary PCs you play regularly
  • PNPC — Personal NPCs under your permanent authorship
  • MSPNPC — Mission-specific PNPCs (temporary)
  • NPC — General non-player characters
  • All — show everyone (default)

Sorting

The Sort dropdown offers four options:

  • Type & Name (default) — sorted by type priority (Primary first, NPC last), then alphabetically within each type
  • Name A–Z — pure alphabetical order
  • Most Sims — characters with the most recorded sims appear first
  • Most Recent — sorted by the date of the most recent sim

The ↺ button in the manifest header forces a full re-scan of your sims and refreshes all statistics. Use this if you've been editing sims while the manifest was open. Statistics also update automatically as you write.

Click + Add to create a new character manually. LCARS also auto-seeds the Manifest when you open it — any character you've marked as yours across your sims will appear automatically.

Character Profile (Right Pane)

Clicking a character in the list opens their profile in a two-column layout: biography and stats on the left, sim history on the right.

View Mode

The default view shows a polished profile card:

  • Avatar — photo or division-coloured initials. Click it to change the photo.
  • Name, Rank & Pips — displayed with the appropriate pip visualization (see below)
  • Type badge — Primary, PNPC, etc.
  • Division, Species, Aliases, Wiki link, Notes
  • Stats grid — My Sims / All Sims / Avg Words / Last Sim date

The sims column lists every sim where this character appears as yours, with title (click to open), mission, posted date, and word count.

Edit Mode

Click ✏ Edit Profile to switch to the edit form. Fields include:

Deleting from the Manifest

The 🗑 Remove button at the bottom of a profile removes the character's manifest entry. It does not affect any sim data — the character's name in existing sims is unchanged.

13. Full-text Search New in v2

Click the 🔍 button at the top of the sidebar to open the search bar. Type a query (minimum 2 characters) to search across all sim titles and body text simultaneously.

Results

Search results replace the normal sidebar navigation. Each result shows:

  • The sim title with matching text highlighted in amber
  • The mission name
  • A short text snippet from the sim body showing the context of the match, with the matching text highlighted

Click any result to open that sim.

Clearing Search

Click the ✕ button in the search bar, or click 🔍 again, to clear the search and restore the normal navigation tree.

14. Sim Templates New in v2

Templates let you save a sim's content and reuse it as a starting point for future sims — useful for recurring scene openings, standard JP headers, or any other content you write repeatedly.

Saving a Template

With a sim open, click ⚙ → ⬡ Templates…. A dialog appears with an option to save the current sim's content as a new template. Enter a name and click Save as Template.

Using a Template

When creating a new sim via + Sim, the New Sim dialog includes a From Template dropdown. Choose a saved template to pre-fill the new sim's editor with that content. You can then edit it freely — the template itself is not modified.

Managing Templates

Open ⚙ → ⬡ Templates… to view all saved templates. From there you can rename or delete any template.

15. Revision Snapshots New in v2

LCARS automatically saves revision snapshots of your sims as you write, giving you a safety net if you need to recover an earlier version.

How Snapshots Work

A snapshot is saved automatically each time your sim's word count changes by 100 or more words since the last snapshot. Up to 10 snapshots are stored per sim — when an 11th is created, the oldest is removed.

Viewing & Restoring

Click ⟳ in the sim header (Row 1) to open the revision history panel. It lists all snapshots in reverse chronological order, showing the date saved and word count at the time. Click Restore next to any snapshot to replace the current editor content with that version.

Restoring a snapshot replaces the current content immediately and saves. There is no undo for a restore — consider copying the current text elsewhere first if you are unsure.

16. Display & Preferences

Theme

Click ⚙ and choose a theme:

  • Light — white background, dark text

17. Export & Import

LCARS stores all data in your browser's local storage. To back up your work or move it to another browser, use Export and Import.

Export

Click ⬇ Export. A file named lcars-YYYY-MM-DD.lcars is downloaded. It contains all your missions, scenes, sims, characters, templates, and settings in JSON format.

Import

Click ⬆ Import and select a .lcars file. A dialog offers:

  • Cancel — close the dialog, import nothing
  • ⚠ Overwrite — replace all current data with the imported file
  • Merge — add the imported missions, scenes, and sims alongside your existing data
Character Manifest data (including aliases and type assignments), templates, and revision snapshots are all included in the export file and will be fully restored on import.
